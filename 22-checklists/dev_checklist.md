# Developer "Definition of Done" Checklist

## Before Coding
- [ ] Is the feature documented in `01-PRODUCT`?
- [ ] Is the API schema updated in `04-BACKEND`?
- [ ] Are the security implications reviewed (`06-SECURITY`)?

## During Implementation
- [ ] Follows Clean Architecture / DDD patterns.
- [ ] Uses standard naming conventions.
- [ ] No hardcoded secrets or environment-specific URLs.
- [ ] Proper error handling implemented.

## Before Merge
- [ ] Unit tests pass.
- [ ] Linting and formatting rules met.
- [ ] Documentation updated to reflect implementation details.
- [ ] Performance budget not exceeded.
